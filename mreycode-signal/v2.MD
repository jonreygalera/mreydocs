## ✨ Key Features

- 🍱 **Bento Grid Architecture**: Beautiful, fully responsive grid layout that elegantly adapts across mobile, tablet, and desktop viewports.
- 📺 **TV / Kiosk Mode**: A distraction-free, full-screen mode perfect for office displays, command centers, and NOC dashboards.
- 🗄️ **Supabase Integration**: Seamlessly sync, persist, and retrieve your dashboard configurations via Supabase, enabling cloud-stored widget settings across devices.
- 🔗 **Iframe Embedding**: Embed external dashboards directly into Signal, or host Signal widgets as iframes in your other web properties.
- ⚡ **Real-time Polling**: Configurable refresh intervals ensure your data and charts are always up to the second without manual reloading.
- 🚀 **Webhook Actions**: Fire off external webhooks directly from your dashboard widgets, allowing you to trigger builds, alerts, or automation from a tailored UI.
- 🛠️ **Fast Widget Initialization**: Create, test, and preview new widgets instantly via an intuitive in-app JSON editor and playground.
- 🎨 **Dynamic Vibe Styling**: Accent colors, beautiful glassmorphism, semantic rules based on data thresholds (e.g. green when up, red when down), and rich micro-animations.

## 🛠️ Getting Started & Project Usage

Signal is designed to be effortlessly spun up locally or deployed immediately to your cloud hosting environment.

### Prerequisites
- Node.js (v18+)
- npm or yarn

### Installation

1. **Clone the repository:**
   ```bash
   git clone https://github.com/jonreygalera/mreycode-signal.git
   cd mreycode-signal
   ```

2. **Install dependencies:**
   ```bash
   npm install
   ```

3. **Environment Setup (Optional but recommended for Cloud Sync):**
   Create a `.env.local` file in the root and add your Supabase credentials:
   ```env
   NEXT_PUBLIC_SUPABASE_URL=your-project-url
   NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
   ```

4. **Run the development server:**
   ```bash
   npm run dev
   ```

5. **Open the app:**
   Navigate to `http://localhost:3000` in your web browser.

## 🧩 Usage of the Widget

Signal widgets are fully driven by a structured JSON configuration. This configuration handles metadata (labels, sizing), API specifications (endpoints, methods, intervals, parsing), nested `config` (chart types, iframes, clocks), and semantic visual styling rules.

You can add widgets dynamically via the **Fast Widget Modal** on your dashboard or build templates into the codebase.

### 1. Core Properties Configuration

Every widget requires a standard set of core parameters.

| Key | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `id` | `string` | **Yes** | Unique identifier for the widget instance. |
| `label` | `string` | **Yes** | The display title shown prominently on the card. |
| `type` | `string` | **Yes** | The core visualization framework (`stat`, `chart`, `iframe`, `clock`, `action`). |
| `size` | `string` | No | Width span (`sm`, `md`, `lg`, `xl`). Automatically wraps nicely on UI. |
| `description` | `string` | No | Subtitle shown underneath the label. |

### 2. Data Fetching API

Link widgets to live data endpoints using the API fields.

| Key | Type | Description |
| :--- | :--- | :--- |
| `api` | `string` | Complete endpoint URL, or `'none'` for static widgets. |
| `method` | `GET \| POST` | HTTP request method. Default is `GET`. |
| `headers` | `Record<string, string>` | Required headers (e.g. Authorization, Content-Type). |
| `responsePath` | `string` | Dot-notation string to extract data (e.g., `data.metrics.count`). |
| `refreshInterval` | `number` | Time in milliseconds between polling requests. |
| `transformer` | `string` | **JS Code string** to evaluate and transform data on the fly before render. |

### 3. Nested `config` Details

Widget-specific behavior is housed tightly within the `config` object.

| Key | Applicable Types | Description |
| :--- | :--- | :--- |
| `config.chart` | `chart` | Pick between visual formats (`line`, `bar`, `area`). |
| `config.xKey` & `yKey` | `chart` | Keys mapped to graph X and Y axes. |
| `config.prefix` & `suffix` | `stat`, `chart` | String units applied strictly to value rendering (e.g. `$`, `%`). |
| `config.abbreviate` | `stat`, `chart` | Shorten numbers visually (e.g. 15,200 becomes 15.2k). |
| `config.displayType` | `clock` | Specify either `analog` or `digital`. |
| `config.timezone` | `clock` | Define standard IANA timezones (defaults to local device zone). |
| `config.iframeUrl` | `iframe` | Remote URL target for iframe widgets. |
| `config.actionConfig` | `action` | Setup configuration for webhook action triggers (url, payload template). |

### 4. Semantic Aesthetics

Widgets intuitively react visually to their own data streams securely.

| Key | Type | Description |
| :--- | :--- | :--- |
| `accentColor` | `string` | Explicit HEX value applying glows and focus outlines. |
| `baseColor` | `string` | Overrides the global theme color specifically for this widget. |
| `colorRules` | `Array` | Arrays of evaluation rules (`condition`, `value`, `color`) dynamically resolving to system colors (`up`, `down`, `warning`). |

## 📝 Configuration Examples

### Stat Metric with Transformation
```json
{
  "id": "active-users-stat",
  "label": "Active Users",
  "type": "stat",
  "size": "sm",
  "api": "https://api.example.com/v1/users",
  "method": "GET",
  "responsePath": "data.activeCount",
  "refreshInterval": 60000,
  "config": {
    "abbreviate": true
  },
  "colorRules": [
    { "condition": ">=", "value": 1000, "color": "up" },
    { "condition": "<", "value": 1000, "color": "warning" }
  ]
}
```

### Area Chart Widget
```json
{
  "id": "revenue-trend-lg",
  "label": "Weekly Revenue",
  "type": "chart",
  "size": "lg",
  "api": "https://api.example.com/v1/revenue/weekly",
  "config": {
    "chart": "area",
    "xKey": "date",
    "yKey": "amount",
    "prefix": "$"
  }
}
```

### Webhook Action Trigger
```json
{
  "id": "trigger-build",
  "label": "Deploy Production",
  "type": "action",
  "description": "Trigger Vercel build manually",
  "config": {
    "actionConfig": {
      "url": "https://api.vercel.com/v1/integrations/deploy/PROJECT_MAPPING",
      "method": "POST"
    }
  }
}
```

### Remote Iframe Embed
```json
{
  "id": "grafana-embed",
  "label": "Grafana Server Load",
  "type": "iframe",
  "size": "xl",
  "config": {
    "iframeUrl": "https://grafana.yourdomain.com/d/xyz?orgId=1&theme=dark"
  }
}
```

## 🗄️ Cloud Sync (Supabase integration)

Signal offers optional, zero-configuration database continuity out of the box using **Supabase**.
By connecting your interface to Supabase (from the Settings icon > Connect Database), your dashboard layout, widget settings, and preferences automatically sync to cloud tables.

If you jump across devices (ex. from laptop to an office TV screen), simply provide the keys in the settings modal, and Signal pulls down your exact configuration in real-time.
