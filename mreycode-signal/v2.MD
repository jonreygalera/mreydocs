# mreycode-signal (v2.0.0)

A high-performance, bento-style metrics dashboard designed for elegance and real-time visibility. Built with a **Vibe Coding** philosophy using the **Google Antigravity IDE**, Signal provides a unified platform for monitoring APIs, external dashboards, and system metrics through a flexible, JSON-driven architecture.

## 🚀 Version 2.0.0 Highlights

The v2.0.0 release introduces a significant architectural refactor to improve scalability and developer experience:

- **Nested Configuration**: Widget-specific settings are now isolated in a dedicated `config` object, keeping the top-level schema clean and consistent.
- **Consolidated Chart Engine**: A unified `chart` type that supports `line`, `bar`, and `area` visualizations via a simple configuration switch.
- **Centralized Documentation**: A single source of truth for the widget dictionary, shared between the [In-App Docs](/docs) and the [Fast Widget Modal].
- **Local Storage Migration**: Automatic conversion of legacy (v1.x) widget configurations to the new v2.0 standard.

## 📖 Configuration Guide

Signal widgets are defined using a structured JSON object. The configuration is split into **Core Properties** and a nested **Widget Config**.

### 1. Core Properties

| Key           | Type     | Description                                                 | Required |
| :------------ | :------- | :---------------------------------------------------------- | :------- |
| `id`          | `string` | Unique identifier for the widget instance.                  | Yes      |
| `label`       | `string` | The display title shown on the card.                        | Yes      |
| `type`        | `string` | The visualization engine (`stat`, `chart`, `iframe`, etc.). | Yes      |
| `size`        | `string` | Horizontal width (`sm`, `md`, `lg`, `xl`).                  | No       |
| `description` | `string` | Sub-text shown under the label.                             | No       |

### 2. Data Fetching

| Key               | Type              | Description                                       |
| :---------------- | :---------------- | :------------------------------------------------ |
| `api`             | `string`          | Endpoint URL or `'none'` for static widgets.      |
| `method`          | `'GET' \| 'POST'` | HTTP request method.                              |
| `responsePath`    | `string`          | Dot-notation path to data (e.g., `result.count`). |
| `refreshInterval` | `number`          | Polling time in ms (min 30000).                   |
| `transformer`     | `string`          | Optional JS code to transform data on the fly.    |

### 3. Nested `config` Object

| Key                      | Type      | Description                                        |
| :----------------------- | :-------- | :------------------------------------------------- |
| `config.chart`           | `string`  | Style selector for charts (`line`, `bar`, `area`). |
| `config.xKey / yKey`     | `string`  | Data keys for chart axes.                          |
| `config.prefix / suffix` | `string`  | Units to prepend or append to values.              |
| `config.abbreviate`      | `boolean` | Shorten large numbers (e.g. 1.2M).                 |
| `config.timezone`        | `string`  | IANA Timezone for Clock widgets.                   |
| `config.displayType`     | `string`  | `analog` or `digital` for Clock widgets.           |
| `config.iframeUrl`       | `string`  | External URL for the Iframe engine.                |

### 4. Aesthetics

| Key           | Type     | Description                                                    |
| :------------ | :------- | :------------------------------------------------------------- |
| `accentColor` | `string` | Custom glow/border color (HEX).                                |
| `color`       | `string` | Static theme color (`up`, `down`, `warning`, `info`, `muted`). |
| `colorRules`  | `object` | Dynamic color thresholds based on data values.                 |

## 📝 Widget Examples

### Stat Widget

```json
{
  "id": "tpl-stat-vibe",
  "type": "stat",
  "label": "Total Vibe",
  "api": "/api/vibe",
  "config": {
    "prefix": "$",
    "suffix": "!",
    "abbreviate": true
  },
  "color": "info"
}
```

### Area Chart Widget

```json
{
  "id": "revenue-trend",
  "type": "chart",
  "label": "Revenue Trend",
  "api": "/api/revenue",
  "size": "lg",
  "config": {
    "chart": "area",
    "xKey": "day",
    "yKey": "value",
    "prefix": "$"
  }
}
```

### World Clock

```json
{
  "id": "manila-clock",
  "type": "clock",
  "label": "Manila Time",
  "config": {
    "displayType": "digital",
    "timezone": "Asia/Manila"
  }
}
```

## 🧠 Advanced Configuration

### Data Transformers

Signal allows you to process raw API data using JavaScript middleware. Define a function body in the `transformer` field:

- **Math**: `(val) => val * 100`
- **Formatting**: `(val) => val.toLocaleString()`
- **List Mapping**: `(val) => val.map(i => ({ name: i.label, value: i.count }))`

### Color Rules

Set dynamic semantic colors based on thresholds:

- `aboveZero` / `belowZero`: Basic coloring.
- `rules`: An array of `condition`, `value`, and `color`.

## 💅 Vibe Coding Tips

1. **Prioritize the Glow**: Use `accentColor` to create depth and focus.
2. **Abbreviate Aggressively**: Keep the dashboard clean by shortening large numbers in `config`.
3. **Use the Fast Widget**: Rapidly prototype new metrics before hardware-coding them.
