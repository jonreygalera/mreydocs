## ✨ Key Features

- 🍱 **Bento Grid Architecture**: Beautiful, fully responsive grid layout that elegantly adapts across mobile, tablet, and desktop viewports.
- 📺 **TV / Kiosk Mode**: A distraction-free, full-screen mode perfect for office displays, command centers, and NOC dashboards.
- 🗄️ **Supabase Integration**: Seamlessly sync, persist, and retrieve your dashboard configurations via Supabase, enabling cloud-stored widget settings across devices.
- 🔗 **Iframe Embedding**: Embed external dashboards directly into Signal, or host Signal widgets as iframes in your other web properties.
- ⚡ **Real-time Polling**: Configurable refresh intervals ensure your data and charts are always up to the second without manual reloading.
- 🚀 **Webhook Actions**: Fire off external webhooks directly from your dashboard widgets, allowing you to trigger builds, alerts, or automation from a tailored UI.
- 🛠️ **Fast Widget & Playground**: Create, test, and preview new widgets instantly via an intuitive in-app JSON editor and a powerful Autocomplete Playground.
- 🎨 **Dynamic Vibe Styling**: Accent colors, beautiful glassmorphism, semantic rules based on data thresholds (e.g. green when up, red when down), and rich micro-animations.

## 🛠️ Getting Started & Project Usage

Signal is designed to be effortlessly spun up locally or deployed immediately to your cloud hosting environment.

### Prerequisites
- Node.js (v18+)
- npm or yarn

### Installation

1. **Clone the repository:**
   ```bash
   git clone https://github.com/jonreygalera/mreycode-signal.git
   cd mreycode-signal
   ```

2. **Install dependencies:**
   ```bash
   npm install
   ```

3. **Environment Setup (Optional but recommended for Cloud Sync):**
   Create a `.env.local` file in the root and add your Supabase credentials:
   ```env
   NEXT_PUBLIC_SUPABASE_URL=your-project-url
   NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
   ```

4. **Run the development server:**
   ```bash
   npm run dev
   ```

5. **Open the app:**
   Navigate to `http://localhost:3000` in your web browser.

## 🧩 Usage of the Widget

Signal widgets are fully driven by a structured JSON configuration. This configuration handles metadata (labels, sizing), API specifications (endpoints, methods, intervals, parsing), nested `config` (chart types, iframes, clocks), semantic visual styling rules, and alerting signals.

### 1. Core Properties Configuration

Every widget requires a standard set of core parameters.

| Key | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `id` | `string` | **Yes** | Unique identifier for the widget instance. |
| `label` | `string` | **Yes** | The display title shown prominently on the card. |
| `type` | `string` | **Yes** | Core visualization mapping (see Available Widget Types). |
| `size` | `string` | No | Width span (`sm`, `md`, `lg`, `xl`, `2xl`, `3xl`). Automatically wraps nicely on UI. |
| `description` | `string` | No | Subtitle shown underneath the label. |
| `api` | `string` | No | Complete endpoint URL, or `'none'` for static widgets. |
| `method` | `GET \| POST` | No | HTTP request method. Default is `GET`. |
| `refreshInterval` | `number` | No | Time in milliseconds between polling requests. |
| `transformer` | `string` | No | **JS Code string** to evaluate and transform data before render. |

### 2. Available Widget Types & Their `config`

Widget-specific behavior is housed tightly within the nested `config` object depending on the `type` assigned.

#### `stat`
A single large statistic or metric.
- `config.prefix`: String prepended to the number (e.g. `$`).
- `config.suffix`: String appended to the number (e.g. `%`).
- `config.abbreviate`: Boolean to shorten large numbers (e.g. 15,200 becomes 15.2k).

#### `chart`
Visualized data via a chart engine.
- `config.chart`: Visual format (`line`, `bar`, `area`).
- `config.xKey`: Key mapped to the X-axis.
- `config.yKey`: Key mapped to the Y-axis.
- `config.prefix` & `config.suffix`: Similar to stat widgets to decorate chart popups/axes.

#### `clock`
A localized timezone clock.
- `config.displayType`: Choice between `analog` or `digital`.
- `config.timezone`: Standard IANA timezone string (defaults to local).

#### `iframe`
Used to embed external contents seamlessly inside the dashboard.
- `config.iframeUrl`: Remote URL target for the iframe.

#### `progress`
A visual progress bar tracking fractional values against a maximum.
- `config`: Handles bar specific themes natively.

#### `status`
A boolean active/inactive status block, great for system health checks.
- `config.trueLabel` & `config.falseLabel`: Overrides for the boolean text outputs.
- `config.trueIcon` & `config.falseIcon`: Overrides for the visual icons representing status.

#### `label`
Distinct textual layout blocks without heavy metrics.
- `config.subtitle`: Accompanying text.
- `config.align`: Alignment rules (`left`, `center`, `right`).
- `config.variant`: Sizing modes (`simple`, `pill`, `hero`).
- `config.link`: Hyperlinks.

#### `list`
Displays iterable arrays of data directly via list mapping.

#### `pulse`
Visually pulsing indicator lights natively wired to activity.
- `config.pulseSpeed`: Speed interval (`slow`, `normal`, `fast`).
- `config.valueLabel`: Internal text inside the pulsing orb.

## 🚨 Alerts & Signals Setup

Every widget can listen to its numeric data output and fire **Signals**. Signals evaluate custom conditions against the payload and can execute various side-effects.

### Signal Payload
A signal is an object array added entirely onto a widget property called `signals`.

| Key | Type | Description |
| :--- | :--- | :--- |
| `condition` | `string` | The evaluation rule: `above`, `below`, `equals`, `diff`. |
| `threshold` | `number` | The numerical trigger limit. |
| `action` | `string[]` | Array of actions: `notify`, `pulse`, `sound`, `notify-in-app`, `webhook`. |
| `enabled` | `boolean` | Allows for easily turning alerts off globally for the signal. |
| `cooldown` | `number` | Cooldown period between sequential fires (minutes). |
| `webhook` | `object` | Detailed webhook settings required if `webhook` action is chosen. |

### Webhook Configuration
If your signal includes `webhook` in its `action` array, you seamlessly power it with the `webhook` object:

```json
"webhook": {
  "url": "https://hooks.slack.com/...",
  "method": "POST",
  "headers": {
    "Content-Type": "application/json"
  },
  "body": "{ \"text\": \"Threshold exceeded!\" }"
}
```

## 🕹️ The Playground & Templates

Building and troubleshooting JSON can be tricky. Signal solves this with the built-in **Playground**.

- **Autocomplete Template Browser**: The playground features an insanely fast autocomplete dropdown that scales identically over **10,000+ templates**. Just search to grab configurations pre-baked for dozens of public services and dashboard views.
- **Live Preview Environment**: You can rapidly edit a widget configuration on the left-side editor, and see a hyper-live rendered view of the widget immediately on the right side.
- **Fast Widget Modal**: In your main dashboard, hitting "Add Widget" will open up the exact same intuitive form builder, minimizing typing overhead.

## 🗄️ Cloud Sync (Supabase integration)

Signal offers optional, zero-configuration database continuity out of the box using **Supabase**.
By connecting your interface to Supabase (from the Settings icon > Connect Database), your dashboard layout, widget settings, and preferences automatically sync to cloud tables.

If you jump across devices (ex. from laptop to an office TV screen), simply provide the keys in the settings modal, and Signal pulls down your exact configuration in real-time.
